# GitHub Branding Options

## Recommended

- Repo name: `ai-alpha-build-log`
- GitHub description: `AI-native full-stack blog and build log showing how I use AI Alpha workflows to ship faster for recruiters, clients, and real production work.`
- Why this works: `AI Alpha` stays front and center, while `build log` sounds credible and technical instead of hype-heavy.

## Strong alternatives

### 1. `ai-alpha-dev-blog`

- Description: `A full-stack engineering blog documenting how AI Alpha workflows help me move faster across frontend, backend, content systems, and client delivery.`

### 2. `ai-alpha-workbench`

- Description: `My AI-native full-stack workbench: blog, experiments, and production-style builds shaped by high-output developer workflows.`

### 3. `ai-alpha-shipping-log`

- Description: `A technical blog and shipping log showing how I use AI Alpha to compress research, design, coding, and delivery cycles.`

### 4. `ai-alpha-stack`

- Description: `A full-stack blog built to show recruiters and freelance clients how I combine AI Alpha workflows with real product engineering.`

### 5. `ai-alpha-systems`

- Description: `Technical writing and full-stack build notes on the systems, tooling, and habits behind my AI Alpha productivity edge.`

### 6. `ai-alpha-output`

- Description: `An AI-native dev blog focused on output: full-stack execution, technical writing, and the workflows that let me ship at a higher rate.`

## Best positioning note

- Do not put `10x` directly in the repo name. It reads louder and less credible than the work itself.
- Put the `10x productivity` idea in the description, README, and pinned-repo framing instead.

## Best recruiter/client-friendly version

- Repo name: `ai-alpha-build-log`
- GitHub description: `AI-native full-stack blog and build log showing how I use AI Alpha workflows to increase output, ship faster, and deliver stronger client-ready product work.`
